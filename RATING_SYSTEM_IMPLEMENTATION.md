## Обзор системы

Система рейтингов позволяет отслеживать результаты игроков и команд в страйкбольных событиях, рассчитывать статистику (игры, победы, точность, K/D ratio) и формировать рейтинговые таблицы.

### Основные возможности:
- ✅ Запись результатов событий (индивидуальные и командные)
- ✅ Автоматический расчет очков на основе места, убийств и точности
- ✅ Обновление статистики игроков и команд
- ✅ Рейтинговые таблицы (лидерборды)
- ✅ Подтверждение результатов (статусы PENDING → CONFIRMED)
- ✅ Автоматический расчет командных результатов на основе результатов игроков
- ✅ Отображение рейтингов в профиле пользователя
- ✅ Управление рейтингами через админ-панель

### 1. База данных (Prisma Schema)

#### Новые модели:

**`EventResult`** - Результаты событий
```prisma
model EventResult {
  id          Int              @id @default(autoincrement())
  eventId     Int
  userId      Int?
  teamId      Int?
  placement   EventPlacement
  points      Int
  kills       Int?
  deaths      Int?
  accuracy    Decimal?
  status      EventResultStatus @default(PENDING)
  confirmedAt DateTime?
  confirmedBy Int?
  createdAt   DateTime         @default(now())
  updatedAt   DateTime         @updatedAt
  
  event       Event            @relation(fields: [eventId], references: [id])
  user        User?            @relation(fields: [userId], references: [id])
  team        Team?            @relation(fields: [teamId], references: [id])
}
```


## Что было изменено и почему

### 1. Система прав доступа

**Было:** Использовался декоратор `@Admin()` для всех операций с рейтингами

**Стало:** Используется `@Application('eventId', true)` для проверки прав владельца события

**Почему:** Владельцы событий должны иметь возможность управлять рейтингами своих событий, а не только системные администраторы.

### 2. Логика отображения событий в админ-панели

**Было:** События фильтровались только по `isActive`

**Стало:** 
- Вкладка "Страйкбольні ігри": показывает все незавершенные события (даже если дата прошла)
- Вкладка "Очікують результатів": показывает только завершенные события, где не все участники имеют результаты
- Вкладка "Архів": показывает завершенные события с результатами

**Почему:** Нужна четкая категоризация событий по статусу и необходимости ввода результатов.

### 3. Управление рейтингами для командных событий

**Было:** Не было специальной логики для командных событий

**Стало:** 
- Для командных событий: выбор команды → выбор игрока из команды
- Автоматический расчет результата команды на основе результатов игроков
- Группировка результатов по командам при отображении

**Почему:** Для командных событий нужен другой UX - сначала выбрать команду, потом игрока из этой команды. Также нужно автоматически рассчитывать результат команды.

### 4. Статусы результатов

**Было:** Результаты сразу применялись к статистике

**Стало:** Результаты создаются со статусом `PENDING`, статистика обновляется только после подтверждения (`CONFIRMED`)

**Почему:** Нужна возможность проверить результаты перед их применением к статистике. Это позволяет исправлять ошибки без влияния на рейтинги.

### 5. Отображение статистики в профиле

**Было:** Использовались моки из `UserData.json`

**Стало:** Данные всегда берутся из API через `getMyPlayerRating()`

**Почему:** Пользователи должны видеть свою реальную статистику, рассчитанную на основе их результатов в событиях.

### 6. Расчет очков

**Формула:**
```
Очки = Базовые очки за место + (Kills × 2) + (Accuracy × 0.5)

Базовые очки:
- FIRST: 100
- SECOND: 75
- THIRD: 50
- PARTICIPATED: 25
```

**Почему:** Нужна справедливая система оценки, учитывающая не только место, но и индивидуальную производительность (убийства и точность).

### 7. Расчет статистики

**Accuracy (Точность):** Среднее значение по всем результатам
**K/D Ratio:** Общее количество kills / общее количество deaths
**Win Rate:** (Победы / Всего игр) × 100%

**Почему:** Эти метрики дают более полную картину производительности игрока, чем просто количество игр и побед.

## Полный флоу работы системы

### 1. Создание события

1. Владелец события создает событие через админ-панель
2. Событие создается со статусом `isActive: true`, `isCompleted: false`
3. Пользователи регистрируются на событие
4. Владелец одобряет регистрации

### 2. Проведение события

1. Событие проходит в указанную дату
2. После окончания события владелец нажимает "Завершити подію"
3. Событие получает статус `isCompleted: true`, `completedAt: <дата>`
4. Событие появляется во вкладке "Очікують результатів"

### 3. Ввод результатов

#### Для индивидуальных событий:

1. Владелец открывает событие из вкладки "Очікують результатів"
2. Нажимает "Додати рейтинги"
3. Выбирает участника из списка зарегистрированных
4. Вводит данные:
   - Место (FIRST, SECOND, THIRD, PARTICIPATED)
   - Kills (опционально)
   - Deaths (опционально)
   - Accuracy (опционально)
5. Система автоматически рассчитывает очки
6. Результат создается со статусом `PENDING`
7. Статистика игрока НЕ обновляется

#### Для командных событий:

1. Владелец открывает событие
2. Нажимает "Додати рейтинги"
3. Выбирает команду из списка зарегистрированных команд
4. Выбирает игрока из выбранной команды
5. Вводит данные (место, kills, deaths, accuracy)
6. Результат создается со статусом `PENDING` для игрока
7. Система автоматически пересчитывает результат команды на основе всех результатов игроков этой команды
8. Статистика НЕ обновляется

### 4. Подтверждение результатов

1. Владелец проверяет все введенные результаты
2. Нажимает "Підтвердити всі результати" (доступна только если все участники имеют результаты)
3. Для каждого результата:
   - Статус меняется на `CONFIRMED`
   - Устанавливается `confirmedAt` и `confirmedBy`
   - Вызывается `updateRatingsFromResult()`:
     - Обновляется статистика игрока (`PlayerStats`)
     - Обновляется статистика команды (если это командное событие)
     - Пересчитывается ранг игрока/команды
4. Статистика применяется к рейтингам

### 5. Отображение результатов

#### В админ-панели:

1. На странице события отображаются все результаты
2. Результаты группируются:
   - Для командных событий: по командам
   - Для индивидуальных событий: по сторонам (если есть)
   - Если нет группировки: простой список
3. Показывается статус каждого результата (PENDING/CONFIRMED)

#### На публичном сайте:

1. На странице события (`/events/[id]`) отображается секция "Результати події"
2. Показываются только подтвержденные результаты
3. Результаты отсортированы по месту и очкам

### 6. Отображение статистики в профиле

1. Пользователь открывает свой профиль (`/profile`)
2. `ProfileHeader` вызывает `fetchUser()` из `userStore`
3. `userStore` делает параллельные запросы:
   - `getUser()` - данные пользователя
   - `getMyTeams()` - команды пользователя
   - `getMyPlayerRating()` - рейтинг пользователя
4. Статистика формируется из данных рейтинга:
   - `gamesPlayed` - количество игр
   - `wins` - победы
   - `accuracy` - средняя точность (в процентах)
   - `kdRatio` - общий K/D ratio
5. Отображается в виде карточек статистики

### 7. Отображение матчей в профиле

1. `MatchList` компонент загружает результаты пользователя через `getUserEventResults()`
2. Фильтруются только подтвержденные результаты (`status: CONFIRMED`)
3. Результаты преобразуются в формат матчей:
   - Дата из `confirmedAt` или `createdAt`
   - Название события из `event.name`
   - Результат определяется по `placement`:
     - `FIRST` → "Перемога"
     - `SECOND`, `THIRD` → "Учасник"
     - `PARTICIPATED` → "Учасник"
   - K/D из `kills` и `deaths`
4. Сортируются по дате (новые сначала)
5. Показываются последние 10 матчей

### 8. Рейтинговые таблицы

1. Пользователь открывает страницу рейтингов (`/ratings`)
2. Загружаются лидерборды:
   - `getPlayerLeaderboard()` - топ игроков
   - `getTeamLeaderboard()` - топ команд
3. Данные сортируются по `totalPoints` (по умолчанию)
4. Отображаются в виде таблиц с рангами, именами, статистикой

### 9. Редактирование результатов

1. Владелец может редактировать результат до подтверждения
2. При редактировании:
   - Обновляются данные результата
   - Пересчитываются очки
   - Если это командное событие, пересчитывается результат команды
3. После подтверждения редактирование недоступно (можно только удалить и создать заново)

### 10. Удаление результатов

1. Владелец может удалить результат
2. Если результат был подтвержден:
   - Откатывается статистика игрока/команды (`rollbackStatsFromResult()`)
   - Удаляется результат
3. Если результат был PENDING:
   - Просто удаляется
4. Если это командное событие и удаляется результат игрока:
   - Пересчитывается результат команды

## База данных

### Схема связей

```
Event
├── EventResult (один ко многим)
│   ├── User (многие к одному, опционально)
│   └── Team (многие к одному, опционально)
│
User
├── PlayerStats (один к одному)
└── EventResult (один ко многим)

Team
├── TeamStats (один к одному)
└── EventResult (один ко многим)
```

### Индексы

- `EventResult`: `eventId`, `userId`, `teamId`, `status`
- `PlayerStats`: `userId` (unique)
- `TeamStats`: `teamId` (unique)

### Ограничения

- `EventResult`: Либо `userId`, либо `teamId` должен быть установлен (но не оба одновременно для одного результата)
- `EventResult`: Уникальность `(eventId, userId)` для индивидуальных результатов
- `EventResult`: Уникальность `(eventId, teamId)` для командных результатов

## Формулы и расчеты

### Расчет очков

```typescript
function calculatePoints(placement, kills, accuracy) {
  const basePoints = {
    FIRST: 100,
    SECOND: 75,
    THIRD: 50,
    PARTICIPATED: 25
  };
  
  let points = basePoints[placement] || 0;
  
  if (kills) {
    points += kills * 2; // +2 очка за каждое убийство
  }
  
  if (accuracy) {
    points += Math.round(accuracy * 0.5); // +0.5 очка за каждый процент точности
  }
  
  return points;
}
```

### Расчет статистики игрока

```typescript
// При добавлении результата:
gamesPlayed += 1;
totalPoints += result.points;
averagePoints = totalPoints / gamesPlayed;

if (placement === FIRST) {
  wins += 1;
} else if (placement === SECOND || placement === THIRD) {
  losses += 1;
} else {
  draws += 1;
}

winRate = (wins / gamesPlayed) * 100;

// Accuracy - среднее по всем результатам
accuracy = sum(allResults.accuracy) / count(allResults.accuracy);

// K/D Ratio - общее количество kills / общее количество deaths
kdRatio = totalKills / totalDeaths;
```

### Расчет ранга

Ранги пересчитываются на основе `totalPoints`:
1. Все игроки/команды сортируются по `totalPoints` (по убыванию)
2. Присваивается ранг на основе позиции в отсортированном списке
3. Сохраняется `previousRank` для отслеживания изменений

## Безопасность

### Права доступа

- **Чтение рейтингов**: Публичный доступ (через API key)
- **Управление рейтингами**: Только владельцы событий (через ACL)
- **Получение своего рейтинга**: Требует авторизации

### Валидация

- Все входные данные валидируются через DTOs
- Проверка типов и диапазонов значений
- Проверка существования связанных сущностей (событие, пользователь, команда)
